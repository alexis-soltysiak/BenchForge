**Python solution**

```python
from collections import defaultdict
from typing import List, Dict

def group_anagrams(words: List[str]) -> List[List[str]]:
    """
    Groups a list of words into lists of anagrams.

    Parameters
    ----------
    words : List[str]
        The input words (e.g., ["eat", "tea", "tan", "ate", "nat", "bat"]).

    Returns
    -------
    List[List[str]]
        A list where each sub‑list contains words that are anagrams of each other.
        Order of groups and order inside a group are not important.
    """
    # A dictionary where the key is the canonical representation of an anagram
    # (here we use the sorted tuple of characters) and the value is the list
    # of words that share that representation.
    anagram_map: Dict[tuple, List[str]] = defaultdict(list)

    for word in words:
        # **Canonical key** – characters sorted alphabetically.
        # Using a tuple (immutable) makes it hashable for the dict.
        key = tuple(sorted(word))
        anagram_map[key].append(word)

    # Return only the grouped lists.
    return list(anagram_map.values())


# ----------------------------------------------------------------------
# Example usage
if __name__ == "__main__":
    sample = ["eat", "tea", "tan", "ate", "nat", "bat"]
    print(group_anagrams(sample))
    # Possible output (order may vary):
    # [['eat', 'tea', 'ate'], ['tan', 'nat'], ['bat']]
```

---

### How it works (logic)

1. **Canonical representation** –  
   Two words are anagrams iff they contain exactly the same characters with the same multiplicities.  
   By sorting the characters of a word, we obtain a *canonical* form that is identical for all its anagrams (e.g., `"eat"` → `('a','e','t')`).

2. **Hash map (dictionary)** –  
   We use a `defaultdict(list)` where the sorted‑tuple key points to a list of all words that share that key.  
   While iterating over the input, each word is placed into its corresponding bucket.

3. **Collect the groups** –  
   After processing all words, the dictionary values are the required groups of anagrams.

---

### Time complexity

*Sorting each word:*  
For a word of length `k`, sorting costs `O(k log k)`.  
If `n` is the number of words and `k` is the maximum word length, the total cost for sorting all words is `O(n * k log k)`.

*Dictionary operations:*  
Insertion into a hash map is `O(1)` average, so the overall extra cost is linear `O(n)`.

**Overall:** `O(n * k log k)` time, where `n` = number of words, `k` = max word length.

---

### Space complexity

* The dictionary stores every word once, plus the key (sorted tuple).  
  This is `O(n * k)` characters in total.  
* The list of groups returned also holds the same data.  

Hence, the auxiliary space (excluding the input) is `O(n * k)`.

```python
from collections import defaultdict

def group_anagrams(words: list[str]) -> list[list[str]]:
    groups = defaultdict(list)
    for word in words:
        # Sorted characters act as a canonical key for anagrams
        key = "".join(sorted(word))
        groups[key].append(word)
    return list(groups.values())

# Example usage:
print(group_anagrams(["eat", "tea", "tan", "ate", "nat", "bat"]))
# Output: [['eat', 'tea', 'ate'], ['tan', 'nat'], ['bat']] (order may vary)
```

### 🔍 Logic Explanation
Anagrams contain the exact same characters with the same frequencies. By sorting the characters of each word, we generate a **canonical key**. Words that are anagrams will always produce the same sorted key. We use a hash map (`defaultdict`) to bucket words under their respective keys, then return the grouped values.

### ⏱️ Complexity Analysis
- **Time Complexity:** `O(n * k log k)`  
  where `n` = number of words, `k` = maximum length of a word.  
  Sorting each word takes `O(k log k)`, and we do this for `n` words. Hash map insertions/lookups average `O(k)` per word, which is dominated by the sorting step.
  
- **Space Complexity:** `O(n * k)`  
  We store a key of length `k` for each of the `n` words in the hash map, plus the output list which contains all original words.

💡 **Pro Tip:** For maximum performance, you can replace `"".join(sorted(word))` with a fixed-size frequency tuple (e.g., counts of `'a'` through `'z'`). This reduces time complexity to `O(n * k)` at the cost of slightly more verbose code. The sorting approach is preferred in practice for its simplicity and readability.
