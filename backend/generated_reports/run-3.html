Here's an efficient solution using a hash map (dictionary) to achieve **O(n) time complexity**:

```python
def two_sum(nums, target):
    seen = {}  # Maps number -> index
    
    for i, num in enumerate(nums):
        complement = target - num
        if complement in seen:
            return [seen[complement], i]
        seen[num] = i
        
    return []  # Return empty list if no solution exists
```

### 🔍 How it Works:
1. We iterate through the list once, keeping track of numbers we've already seen in a dictionary (`seen`).
2. For each number, we calculate its `complement` (`target - num`).
3. If the `complement` is already in `seen`, we've found our pair and return their indices.
4. If not, we store the current number and its index in `seen` and continue.

### 📊 Complexity:
- **Time:** `O(n)` → Single pass through the list. Dictionary lookups/insertions are `O(1)` on average.
- **Space:** `O(n)` → In the worst case, we store all `n` elements in the dictionary.

### 💡 Example Usage:
```python
print(two_sum([2, 7, 11, 15], 9))  # Output: [0, 1]
print(two_sum([3, 2, 4], 6))       # Output: [1, 2]
print(two_sum([3, 3], 6))          # Output: [0, 1]
```

### 📝 Notes:
- Assumes exactly one valid solution exists (standard for this problem). If multiple pairs exist, it returns the first one found.
- Handles negative numbers and zero correctly.
- Returns `[]` if no pair sums to `target` (can be changed to raise an exception if preferred).

To implement an LRU Cache with average $O(1)$ complexity, the optimal approach is a **hybrid data structure** consisting of a **Hash Map** and a **Doubly Linked List**.

### The Approach
*   **Hash Map (Dictionary):** Stores the key as the index and a reference to the corresponding node in the linked list as the value. This allows for $O(1)$ access to any node.
*   **Doubly Linked List:** Maintains the order of usage. The **head** of the list represents the Most Recently Used (MRU) item, and the **tail** represents the Least Recently Used (LRU) item.

**Why this works:**
1.  **`get(key)`:** We use the hash map to find the node in $O(1)$. We then move that node to the front of the list to mark it as recently used.
2.  **`put(key, value)`:** If the key exists, we update it and move it to the front. If it is new, we insert it at the front. If the cache is full, we remove the node at the tail (the LRU item) and delete it from the map.

---

### Implementation

```python
class Node:
    """Represents a node in the doubly linked list."""
    def __init__(self, key=0, value=0):
        self.key = key
        self.value = value
        self.prev = None
        self.next = None

class LRUCache:
    def __init__(self, capacity: int):
        self.capacity = capacity
        self.cache = {}  # Hash Map: key -> Node
        
        # Dummy sentinels to simplify edge cases
        self.head = Node() # Points to MRU side
        self.tail = Node() # Points to LRU side
        self.head.next = self.tail
        self.tail.prev = self.head

    def _add_to_head(self, node):
        """Inserts a node immediately after the head (MRU position)."""
        node.prev = self.head
        node.next = self.head.next
        self.head.next.prev = node
        self.head.next = node

    def _remove_node(self, node):
        """Removes a node from its current position."""
        prev_node = node.prev
        next_node = node.next
        prev_node.next = next_node
        next_node.prev = prev_node

    def _move_to_head(self, node):
        """Moves an existing node to the MRU position."""
        self._remove_node(node)
        self._add_to_head(node)

    def _pop_tail(self):
        """Removes and returns the node at the LRU position."""
        res = self.tail.prev
        self._remove_node(res)
        return res

    def get(self, key: int) -> int:
        if key not in self.cache:
            return -1
        
        node = self.cache[key]
        self._move_to_head(node)
        return node.value

    def put(self, key: int, value: int) -> None:
        if key in self.cache:
            node = self.cache[key]
            node.value = value
            self._move_to_head(node)
        else:
            new_node = Node(key, value)
            self.cache[key] = new_node
            self._add_to_head(new_node)
            
            if len(self.cache) > self.capacity:
                lru_node = self._pop_tail()
                del self.cache[lru_node.key]

# --- Usage Example ---
cache = LRUCache(2)
cache.put(1, 1)  # Cache: {1: 1}
cache.put(2, 2)  # Cache: {1: 1, 2: 2}
print(cache.get(1)) # Returns 1
cache.put(3, 3)    # Evicts key 2, Cache: {1: 1, 3: 3}
print(cache.get(2)) # Returns -1 (not found)
```

    *   **`put(key, value)`:** $O(1)$. Dictionary lookup and insertion/deletion are $O(1)$; list manipulation is $O(1)$.
*   **Space Complexity:**

Here’s a standard O(1)-average implementation of an `LRUCache` in Python using:

- a **hash map** (`dict`) for O(1) lookup by key
- a **doubly linked list** to maintain usage order in O(1)

## Approach

We want:

- `get(key)`:
  - return the value if the key exists
  - mark it as **most recently used**
- `put(key, value)`:
  - insert/update the key
  - mark it as **most recently used**
  - if capacity is exceeded, evict the **least recently used** item

### Why these data structures?

#### 1. Hash Map (`dict`)
Maps `key -> node` so we can find cache entries in O(1).

#### 2. Doubly Linked List
Keeps items ordered by recency:
- **head** = most recently used
- **tail** = least recently used

With a doubly linked list, we can:
- remove any node in O(1)
- move any node to the front in O(1)
- remove the least recently used node from the back in O(1)

---

## Python Implementation

```python
class Node:
    def __init__(self, key=0, value=0):
        self.key = key
        self.value = value
        self.prev = None
        self.next = None


class LRUCache:
    def __init__(self, capacity: int):
        self.capacity = capacity
        self.cache = {}

        # Dummy head and tail to simplify insert/remove logic
        self.head = Node()
        self.tail = Node()

        self.head.next = self.tail
        self.tail.prev = self.head

    def _remove(self, node: Node):
        """Remove node from the doubly linked list."""
        prev_node = node.prev
        next_node = node.next
        prev_node.next = next_node
        next_node.prev = prev_node

    def _insert_at_front(self, node: Node):
        """Insert node right after head (most recently used position)."""
        first_real_node = self.head.next

        node.next = first_real_node
        node.prev = self.head

        self.head.next = node
        first_real_node.prev = node

    def get(self, key: int) -> int:
        if key not in self.cache:
            return -1

        node = self.cache[key]

        # Move accessed node to front
        self._remove(node)
        self._insert_at_front(node)

        return node.value

    def put(self, key: int, value: int) -> None:
        if key in self.cache:
            # Update existing node and move it to front
            node = self.cache[key]
            node.value = value
            self._remove(node)
            self._insert_at_front(node)
        else:
            # Add new node
            node = Node(key, value)
            self.cache[key] = node
            self._insert_at_front(node)

            # Evict least recently used if over capacity
            if len(self.cache) > self.capacity:
                lru = self.tail.prev
                self._remove(lru)
                del self.cache[lru.key]
```

Here’s an `LRUCache` implementation in Python with average **O(1)** `get` and `put`.

## Approach

To support **constant-time lookup** and **constant-time eviction of the least recently used item**, we combine:

1. **Dictionary (`dict`)**
   - Maps `key -> node`
   - Gives O(1) access to cache entries

2. **Doubly linked list**
   - Maintains usage order:
     - Most recently used item near the front
     - Least recently used item near the back
   - Allows O(1) insertion and removal of nodes when they are accessed or evicted

### Why doubly linked list?
A singly linked list cannot remove a node in O(1) unless we already know the previous node. With a doubly linked list, each node has `prev` and `next`, so we can move nodes around efficiently.

---

## Python Implementation

```python
class Node:
    def __init__(self, key=None, value=None):
        self.key = key
        self.value = value
        self.prev = None
        self.next = None


class LRUCache:
    def __init__(self, capacity: int):
        self.capacity = capacity
        self.cache = {}  # key -> Node

        # Dummy head and tail to avoid edge-case checks
        self.head = Node()  # Most recently used side
        self.tail = Node()  # Least recently used side
        self.head.next = self.tail
        self.tail.prev = self.head

    def _remove(self, node: Node):
        """Remove node from the doubly linked list."""
        prev_node = node.prev
        next_node = node.next
        prev_node.next = next_node
        next_node.prev = prev_node

    def _add_to_front(self, node: Node):
        """Add node right after head (most recently used position)."""
        first = self.head.next
        node.prev = self.head
        node.next = first
        self.head.next = node
        first.prev = node

    def get(self, key: int) -> int:
        if key not in self.cache:
            return -1

        node = self.cache[key]
        # Move accessed node to front (most recently used)
        self._remove(node)
        self._add_to_front(node)
        return node.value

    def put(self, key: int, value: int) -> None:
        if self.capacity == 0:
            return

        if key in self.cache:
            node = self.cache[key]
            node.value = value
            # Refresh usage
            self._remove(node)
            self._add_to_front(node)
        else:
            if len(self.cache) == self.capacity:
                # Evict least recently used node (node before tail)
                lru = self.tail.prev
                self._remove(lru)
                del self.cache[lru.key]

            new_node = Node(key, value)
            self.cache[key] = new_node
            self._add_to_front(new_node)
```

### Approach
Use the classic **LRU (Least Recently Used) cache** design combining:

1. **Hash map (`dict`)**: maps `key -> node` for **O(1)** access to cache entries.
2. **Doubly linked list**: maintains usage order in **O(1)** time:
   - Most recently used (MRU) near the **head**
   - Least recently used (LRU) near the **tail**
   - On every `get`/`put`, move the corresponding node to the head.
   - When capacity is exceeded, evict the node at the tail.

This guarantees **average O(1)** complexity because both dictionary operations and linked-list node operations (remove/insert) are O(1).

---

### Python Implementation

```python
class LRUCache:
    class _Node:
        __slots__ = ("key", "value", "prev", "next")

        def __init__(self, key=None, value=None):
            self.key = key
            self.value = value
            self.prev = None
            self.next = None

    def __init__(self, capacity: int):
        if capacity <= 0:
            raise ValueError("capacity must be positive")
        self.capacity = capacity
        self.cache = {}  # key -> _Node

        # Sentinel nodes to avoid edge-case handling for empty list / ends.
        self.head = self._Node()  # MRU side
        self.tail = self._Node()  # LRU side
        self.head.next = self.tail
        self.tail.prev = self.head

    def _remove(self, node: _Node) -> None:
        """Remove an existing node from the linked list in O(1)."""
        prev_node = node.prev
        next_node = node.next
        prev_node.next = next_node
        next_node.prev = prev_node

    def _add_to_front(self, node: _Node) -> None:
        """Insert node right after head (mark as most recently used) in O(1)."""
        first = self.head.next
        node.prev = self.head
        node.next = first
        self.head.next = node
        first.prev = node

    def get(self, key: int) -> int:
        """
        Return value associated with key if present; otherwise return -1.
        Marks the key as most recently used.
        """
        node = self.cache.get(key)
        if not node:
            return -1

        # Move to front (MRU)
        self._remove(node)
        self._add_to_front(node)
        return node.value

    def put(self, key: int, value: int) -> None:
        """
        Insert/update key with value.
        If capacity is exceeded, evict least recently used item.
        """
        node = self.cache.get(key)

        if node:
            # Update value and move to front
            node.value = value
            self._remove(node)
            self._add_to_front(node)
        else:
            # Add new node
            new_node = self._Node(key, value)
            self.cache[key] = new_node
            self._add_to_front(new_node)

            # Evict LRU if over capacity
            if len(self.cache) > self.capacity:
                lru = self.tail.prev  # node just before tail sentinel
                self._remove(lru)
                del self.cache[lru.key]
```
